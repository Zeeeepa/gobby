> **Generated:** 2025-12-15 | **Scan Level:** Exhaustive | **Version:** 0.1.0

## Project Overview

**Gobby** is a local-first daemon that unifies AI coding assistants (Claude Code, Gemini CLI, Codex) through a hook interface for session tracking and provides an MCP proxy with progressive tool discovery for efficient access to downstream servers.

### Quick Facts

| Property | Value |
|----------|-------|
| **Type** | Backend + CLI (Daemon) |
| **Language** | Python 3.11+ |
| **Framework** | FastAPI + FastMCP + Click |
| **Database** | SQLite (local-first) |
| **Package Manager** | uv |
| **License** | MIT |

## Quick Start

```bash
# Install
git clone https://github.com/<owner>/gobby.git
cd gobby
uv sync

# Start daemon
uv run gobby start

# Install hooks to your project
cd /path/to/your/project
gobby install

# Check status
gobby status
```

## Key Capabilities

### Multi-CLI Support
- **Claude Code** - Full hook integration
- **Gemini CLI** - Full hook integration
- **Codex CLI** - Notify integration

### Session Management
- Automatic session registration
- Cross-session context handoff
- LLM-powered session summaries
- Title synthesis from prompts

### MCP Proxy
- Progressive tool discovery (list → schema → execute)
- Connection pooling for downstream servers
- Support for HTTP, stdio, WebSocket transports
- Dynamic server add/remove

### LLM Integration
- Multi-provider support (Claude, Codex, Gemini, LiteLLM)
- Subscription-based and BYOK authentication
- Code execution in Claude sandbox
- Tool recommendations
