Metrics & Measurement:

Pattern accuracy measurement:
- Dataset: Labeled test set of 500+ pattern instances with ground truth annotations
- Definition: True positives / (True positives + False positives + False negatives)
- Validation: 5-fold cross-validation with minimum 100 samples per fold
- Recording: Store datasets in `metrics/pattern_accuracy/` with version control

Insight relevance scoring:
- Rubric: 5-point scale (1=irrelevant, 3=partially relevant, 5=highly relevant)
- Annotators: Minimum 2 independent reviewers per insight
- Agreement: Cohen's kappa >= 0.7 required for inter-annotator reliability
- Aggregation: Weighted mean across annotators, normalized to 0-100%
- Recording: Store rubrics and annotations in `metrics/insight_relevance/`

Knowledge retrieval latency:
- Boundary: End-to-end API response time from request receipt to first payload byte
- Test harness: Automated benchmark suite with standardized query set
- Load profile: 10 concurrent requests, 100 sequential queries per test run
- Percentile: Report p95 latency (95th percentile must be < 500ms)
- Recording: Store test procedures and results in `metrics/retrieval_latency/`
